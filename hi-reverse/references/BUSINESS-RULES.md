# Business-Rule Catalog

Consolidate rules that are otherwise scattered across use cases and source conditions.

## Retrieval

1. Extract guards, calculations, limits, permissions, state invariants, monetary rules, ordering constraints, and decision outcomes from traces and evidence.
2. Query Graph-RAG independently for validations, enable/disable conditions, thresholds, mode gates, and exceptional overrides.
3. Verify exact operators, constants, and branches with Serena.
4. Express multi-condition behavior as decision tables.
5. Link every rule to affected use cases and code evidence.

Write `business-rules.md`. Do not convert a one-time implementation detail into a business rule without repeated or explicit behavioral evidence.
